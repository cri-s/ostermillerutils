CSV Parser

Java's StringTokenizer does not make it easy to parse files of comma separated values for two reasons. First StringTokenizer doesn't handle empty strings and second it doesn't have a way to easily get Strings in quotes that have commas inside them. This CSV parser takes care of those issues and support line numbering, escape characters, and comments.

There is also a regression test and the expected results of that test available.

This parser was created using JFlex and you will need to have it if you want to modify the parser.

Browser

Allows URLs to be opened in the system browser on Windows and Unix. JavaWorld has a Java Tip which shows how to open a URL in the system browser. That implementation has some serious problems:

  1. Security - You must rely on the Runtime.exec() method to open another program in java. You must stick the URL into the command line that you use to open the browser. If you do not check for it, somebody could craft a URL that messes up the command line. This is possible either by adding whitespace to the url (so it is interpretted as more than one argument) or by prepending the url with a '-' so that it is interpretted not as a url but as an option by the browser. This would be an issue if you were writing an instant messanger and were opening URLs sent to in a special URL field. A malicous hacker could exploit this by sending you a URL allowing them to access or crash your computer.
  2. Out of date - There are a variety of browsers available on Linux. It would be nice to check for something better than Netscape. (I prefer Mozilla)
  3. Blocking - If you are using Netscape under Linux, a Java program using this library will wait for netscape to close before continuing.

All of these problems are solved in the com.Ostermiller.util.Browser implementation.
